### 1 상속과 합성의 비교

상속 Inheritance 는 부모 클래스와 자식 클래스를 연결하여 코드를 재사용하는 방법이다 is a 관계이다.
의존성이 컴파일 타임에 해결된다. 부모 클래스의 내부 구현을 알아야 하므로 결합도가 높아지고 캡슐화가 약화된다. 타입 계층을 구조화하는 데 목적이 있다.

합성 Composition 은 전체를 표현하는 객체가 부분을 표현하는 객체를 포함하여 코드를 재사용하는 방법이다 has a 관계이다.
의존성이 런타임에 해결된다. 포함된 객체의 퍼블릭 인터페이스에만 의존하므로 구현 변경의 영향을 최소화한다.

---

### 2 합성이 유연한 이유

캡슐화 유지. 합성은 내부 객체의 구현을 몰라도 인터페이스를 통해 협력하므로 캡슐화가 유지된다.

동적 변경 가능. 상속은 정적인 관계이지만 합성은 실행 시점에 협력 대상을 교체할 수 있어 유연하다.

낮은 결합도. 클래스 수 증가 문제를 완화하고 객체 간 결합도를 낮춰 변경에 강한 구조를 만든다.

---

### 3 상속의 문제점. 조합의 폭발 Combinatorial Explosion

상속으로 여러 기능을 조합하면 가능한 조합 수만큼 클래스가 증가하는 문제가 발생한다.

예를 들어 기본 요금제에 세금 정책 할인 정책 등을 추가할 경우 상속을 사용하면 중복 코드가 늘어나고 계층이 복잡해진다. 유지보수가 어려워진다.

합성을 사용하면 각 정책을 독립 객체로 분리하고 런타임에 조합하여 이 문제를 해결한다.

---

### 4 믹스인 Mixin

믹스인은 객체 생성 시 코드 일부를 클래스에 섞어 넣어 재사용하는 기법이다.
고정된 계층 없이 기능을 재사용할 수 있다.

1. 인터페이스와 디폴트 메서드 Java 8 이상
   인터페이스에 구현을 포함하여 특정 행동을 정의한다. 구현 클래스가 이를 재사용한다.

2. 합성과 포워딩 Forwarding
   기능을 별도 객체로 분리하고 필요한 클래스에 포함한다.
   외부 호출은 포워딩 메서드를 통해 내부 객체에 위임한다.
   퍼블릭 인터페이스에만 의존하므로 캡슐화를 유지한다.

3. 디자인 패턴 기반 조합
   데코레이터 패턴과 전략 패턴으로 믹스인 효과를 구현한다.
   상속 기반 구조의 클래스 증가 문제를 해결하고 실행 시점에 기능을 조합한다.

---

### 포워딩 Forwarding 과 포워딩 메서드

합성 구조에서는 내부 객체가 실제 기능을 수행한다. 외부에서는 이를 알 수 없다.

외부에서 호출이 들어오면 포워딩 메서드가 내부 객체에 요청을 전달한다.

예시. 스마트폰.전화걸기 호출 시 내부 phone.전화걸기 를 실행한다.

외부에서는 내부 구현을 알 필요가 없다.
